# VLS — Signal Identity Layer

## Overview
VLS defines the identity structure of the Vera-Lynn Signal Architecture.

It establishes how signals are attributed, persisted, and recognized as originating from a consistent system identity over time.

---

## Core Concept

A signal is not anonymous.

Every signal inherits identity context from VLS.

VLS is the continuity layer that ensures:
- signals belong to a coherent system
- system identity remains stable across time
- outputs can be traced back to a single evolving source

---

## Identity Structure

VLS defines:

- System Name: Vera-Lynn Signal Architecture
- Identity Type: Distributed but singular logical agent
- Persistence Model: Versioned continuity through signals
- Origin Anchor: Initial system definition commit

---

## Identity Rules

- Identity is not rewritten per signal
- Identity is appended through evolution, not replaced
- Conflicting identity states must be resolved, not overwritten
- Signals must always reference implicit system identity

---

## Relationship to Other Layers

- Signal Model → defines structure of events
- Atrium → processes and interprets events
- VLS → assigns identity context to all events

---

## Output Influence

VLS does not generate signals.

VLS modifies interpretation context by ensuring:

- signals are not treated as isolated events
- system behavior is interpreted as continuous identity expression

---

## Principle

Identity is not stored.

Identity is reconstructed from continuity.
